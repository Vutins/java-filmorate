## Обзор проекта

**Filmorate** — это социальная сеть для любителей кино, позволяющая пользователям:
- Регистрироваться и просматривать информацию о фильмах
- Ставить лайки фильмам и добавлять друзей
- Получать персонализированные рекомендации
- Читать и оставлять отзывы на фильмы
- Отслеживать активность друзей
- Искать фильмы по различным критериям

Проект реализован как RESTful API с использованием Spring Boot и работает с базой данных PostgreSQL.

## Функциональность

### Пользователи
- Регистрация и аутентификация пользователей
- Управление списком друзей (добавление/удаление)
- Поиск общих друзей между пользователями
- Получение ленты событий пользователя (лайки, дружба, отзывы)
- Получение персонализированных рекомендаций фильмов
- Удаление пользователей

### Фильмы
- Добавление и обновление информации о фильмах
- Получение списка фильмов с пагинацией и фильтрацией
- Поиск фильмов по названию и режиссёрам
- Получение популярных фильмов (топ-N по лайкам)
- Фильтрация популярных фильмов по жанру и году
- Получение общих фильмов между пользователями
- Удаление фильмов

### Отзывы (Reviews)
- Создание, обновление и удаление отзывов
- Лайки и дизлайки отзывов
- Рейтинг полезности отзывов
- Получение отзывов по фильму или всех отзывов с сортировкой по полезности

### Режиссёры
- Управление информацией о режиссёрах
- Получение фильмов режиссёра с сортировкой по лайкам или году выпуска

### Дополнительные функции
- Жанры и MPA-рейтинги (справочники)
- История событий пользователя (фид активности)
- Поиск по названию фильма и имени режиссёра
- Валидация входных данных
- Полноценная обработка ошибок с HTTP статусами

## Запуск проекта

### Требования
- Java 21 или выше
- PostgreSQL 12+
- Maven 3.6+

### Шаги для запуска

1. **Клонировать репозиторий:**
```bash
git clone <repository-url>
cd java-filmorate
```

2. **Настроить базу данных:**
- Создать базу данных в PostgreSQL:
```sql
CREATE DATABASE filmorate;
```

3. **Настроить подключение к БД:**
   Создать файл `application.yml` в `src/main/resources/`:
```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/filmorate
    username: your-username
    password: your-password
    driver-class-name: org.postgresql.Driver
  
  sql:
    init:
      mode: always
      continue-on-error: true

logging:
  level:
    org.springframework.jdbc: DEBUG
```

4. **Запустить приложение:**
```bash
mvn spring-boot:run
```

Приложение будет доступно по адресу: `http://localhost:8080`

## REST API Endpoints

### Пользователи

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/users` | Получить всех пользователей |
| GET | `/users/{id}` | Получить пользователя по ID |
| POST | `/users` | Создать нового пользователя |
| PUT | `/users` | Обновить пользователя |
| DELETE | `/users/{userId}` | Удалить пользователя |
| PUT | `/users/{id}/friends/{friendId}` | Добавить друга |
| DELETE | `/users/{id}/friends/{friendId}` | Удалить друга |
| GET | `/users/{id}/friends` | Получить друзей пользователя |
| GET | `/users/{id}/friends/common/{otherId}` | Получить общих друзей |
| GET | `/users/{id}/recommendations` | Получить рекомендации фильмов |
| GET | `/users/{id}/feed` | Получить ленту событий |

### Фильмы

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/films` | Получить все фильмы |
| GET | `/films/{id}` | Получить фильм по ID |
| POST | `/films` | Создать новый фильм |
| PUT | `/films` | Обновить фильм |
| DELETE | `/films/{filmId}` | Удалить фильм |
| PUT | `/films/{id}/like/{userId}` | Поставить лайк фильму |
| DELETE | `/films/{id}/like/{userId}` | Удалить лайк фильму |
| GET | `/films/popular` | Получить популярные фильмы |
| GET | `/films/director/{directorId}` | Получить фильмы режиссёра |
| GET | `/films/common` | Получить общие фильмы пользователей |
| GET | `/films/search` | Поиск фильмов |

### Отзывы

| Метод | Endpoint | Описание |

## Обработка ошибок

API возвращает стандартные HTTP статусы и структурированные сообщения об ошибках:

- **400 Bad Request** - ошибки валидации
- **404 Not Found** - ресурс не найден
- **500 Internal Server Error** - серверные ошибки

Пример ответа с ошибкой:
```json
{
  "error": "ERROR[400]: Произошла ошибка ValidationException: ",
  "description": "Название фильма не может быть пустым"
}
```

## Тестирование

Для тестирования API можно использовать:
- **Postman** - коллекция запросов
- **Swagger** - документация API (при подключении springdoc-openapi)
- **cURL** - командные запросы

Пример тестового запроса cURL:
```bash
# Получить всех пользователей
curl -X GET "http://localhost:8080/users"

# Создать нового пользователя
curl -X POST "http://localhost:8080/users" \
  -H "Content-Type: application/json" \
  -d '{"name":"Test User","email":"test@example.com","login":"testuser","birthday":"2000-01-01"}'
```

### Добавление новой функциональности

1. Создать модель в пакете `model`
2. Создать интерфейс хранилища в `storage`
3. Реализовать репозиторий в `dao.repositories`
4. Создать сервис в `service`
5. Добавить контроллер в `controller`
6. Обновить схему БД в `schema.sql`

### Коммиты и ветки

- `main` - стабильная версия
- `develop` - разработка
- Функциональные ветки создаются от `develop`
